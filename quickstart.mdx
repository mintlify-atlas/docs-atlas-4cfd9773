---
title: "Quick Start"
description: "Install ElastomerClient and run your first Elasticsearch query"
---

This guide walks you through installing ElastomerClient, connecting to Elasticsearch, and performing the most common operations: creating an index, indexing documents, and running a search.

<Steps>
  <Step title="Install the gem">
    Install ElastomerClient directly:

    ```bash
    gem install elastomer-client
    ```

    Or add it to your `Gemfile` and run `bundle install`:

    ```ruby Gemfile
    gem "elastomer-client"
    ```

    ```bash
    bundle install
    ```
  </Step>

  <Step title="Connect to Elasticsearch">
    Require the client and create an instance. By default it connects to `http://localhost:9200`.

    ```ruby
    require 'elastomer_client/client'

    client = ElastomerClient::Client.new
    ```

    To connect to a different host or port, pass a URL:

    ```ruby
    require 'elastomer_client/client'

    client = ElastomerClient::Client.new(url: 'http://my-es-host:9200')
    ```

    <Tip>
      Call `client.ping` to verify that your Elasticsearch instance is reachable before proceeding. It returns `true` on success and `false` on any error, making it safe to use in health checks.

      ```ruby
      client.ping  #=> true
      ```
    </Tip>
  </Step>

  <Step title="Create an index">
    Use `client.index` to obtain an `Index` instance scoped to a name, then call `create` with your settings and mappings.

    ```ruby
    index = client.index('books')

    index.create(
      :settings => { 'index.number_of_shards' => 3 },
      :mappings => {
        :_source    => { :enabled => true },
        :properties => {
          :author => { :type => 'keyword' },
          :title  => { :type => 'text' }
        }
      }
    )
    ```

    You can check whether the index exists or delete it:

    ```ruby
    index.exists?  #=> true
    index.delete
    ```
  </Step>

  <Step title="Index a document">
    Use `client.docs` to obtain a `Docs` instance scoped to the index, then call `index` with your document hash. Include `_id` to assign an explicit document ID.

    ```ruby
    docs = client.docs('books')

    docs.index({
      :_id    => 1,
      :author => 'Mark Twain',
      :title  => 'The Adventures of Huckleberry Finn'
    })
    ```
  </Step>

  <Step title="Search documents">
    Pass an Elasticsearch query hash to `docs.search`. The response body mirrors the Elasticsearch JSON response.

    ```ruby
    results = docs.search({ :query => { :match_all => {} } })
    ```
  </Step>

  <Step title="Check cluster health">
    Use the `cluster` accessor to retrieve health and state information:

    ```ruby
    # Summary health status
    client.cluster.health

    # Detailed cluster state
    client.cluster.state

    # All index templates
    client.cluster.templates
    ```
  </Step>
</Steps>
